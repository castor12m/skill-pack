# Handoff — 세션 인수인계

현재 대화의 작업 상태를 저장하여 새 대화에서 이어서 작업할 수 있게 한다.

## 실행 절차

1. **Git 상태 수집** — 아래 정보를 모두 수집한다:
   - 현재 브랜치명
   - `git status` (staged, unstaged, untracked 파일 목록)
   - `git diff --stat` (변경된 파일 요약)
   - `git log --oneline -10` (최근 커밋 10개)

2. **작업 컨텍스트 정리** — 이번 대화에서 수행한 작업을 요약한다:
   - 어떤 작업을 했는지 (완료된 항목, 진행 중인 항목)
   - 수정한 주요 파일 목록과 각 파일에서 변경한 내용 요약
   - 아직 해결하지 못한 이슈나 에러
   - 다음에 이어서 해야 할 작업
   - 사용자가 언급한 중요한 결정 사항이나 선호

3. **활성 태스크 확인** — TaskList로 현재 태스크가 있으면 상태를 포함한다.

4. **핸드오프 파일 작성** — 아래 형식으로 프로젝트의 `.claude/handoff.md`에 저장한다:

```markdown
# Session Handoff

> 생성: {YYYY-MM-DD HH:MM}
> 브랜치: {branch}
> 프로젝트: {working directory}

## 작업 요약
{이번 세션에서 한 일을 간결하게 서술}

## 완료된 작업
- {완료 항목 1}
- {완료 항목 2}

## 진행 중 / 미완료
- {미완료 항목 — 어디까지 했고 무엇이 남았는지}

## 변경된 파일
| 파일 | 변경 내용 |
|------|-----------|
| {path} | {요약} |

## 미해결 이슈
- {이슈 설명}

## 다음 단계
1. {다음에 해야 할 일}
2. {그 다음}

## Git 상태
{git status 출력 요약}

## 참고 사항
{사용자 결정, 선호, 제약 조건 등}
```

5. **사용자에게 안내** — 파일 저장 완료 후, 새 대화에서 `/resume`을 입력하면 이어서 작업할 수 있다고 알려준다.

## 주의사항
- 핸드오프 파일은 항상 덮어쓴다 (이전 핸드오프를 대체).
- 민감 정보(API 키, 토큰 등)는 절대 포함하지 않는다.
- 요약은 간결하되, 새 대화의 Claude가 맥락을 완전히 파악할 수 있을 만큼 충분히 상세하게 작성한다.
